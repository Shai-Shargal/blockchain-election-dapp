Known Bugs & Limitations

1. Questionnaire Anonymity is UI-Only ⚠️

The matched candidate ID is visible on-chain in transaction calldata.

  • UI hides the matched candidate before vote submission
  • After confirmation, the result is revealed
  • This is not cryptographically private — by design
  • True anonymity requires zero-knowledge proofs (beyond scope)

2. Git History Contains Old Secrets (RESOLVED)

  • Private key was committed to blockchain/.env in early commits
  • File is now properly in .gitignore
  • No active risk — credentials are testnet-only
  • Fresh credentials deployed to Sepolia

3. Missing Features (Intentional)

  • No vote reversal or modification (votes are permanent)
  • No multi-election support (single deployment)
  • No DAO voting or delegation (out of scope)

4. Contracts Not Verified on Etherscan (Minor)

Contracts work and tests pass. Verification can be added post-submission.

Smart Contracts Deep Dive

Election.sol — Core Logic

1. Candidate Management

Admin adds candidates with name + 3 position scores (1-5)

2. Merkle Root Storage

Eligible voter addresses hashed and stored on-chain. Voters prove eligibility with Merkle proof.

3. IPFS CID Storage

Voter registry CSV stored on IPFS; CID recorded on-chain for verification.

4. Time-Gated Voting

Admin sets start and end times. Votes rejected before start or after end.

5. Voting Security (CEI Pattern)

✅ Checks: time, candidate, double-vote, Merkle proof
✅ Effects: mark voted, increment vote count
✅ Interactions: mint 10 BAL tokens

BALToken.sol — ERC-20 Rewards

  • Immutable minter address = Election contract only
  • 10 BAL issued per valid vote
  • No other addresses can mint

End-to-End Demonstration

The Complete Flow (8 Steps)

Setup Phase

  1. Admin connects wallet → Admin Dashboard appears
  2. Admin adds 3 candidates with positions
  3. Admin uploads voter CSV → Merkle root + CID stored on-chain
  4. Admin sets election window (start + 3600 seconds)

Voting Phase

  1. Voter 1 selects candidate directly → votes, receives 10 BAL
  2. Voter 2 uses questionnaire → system matches candidate, votes, receives 10 BAL

Results Phase

  1. Election ends → voting window closes
  2. Results page shows: winner banner + bar chart + ranked table

Key Features Demonstrated
